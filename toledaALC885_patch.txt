The toledaALC885 patch configures OS X Realtek ALC885 onboard audio on Intel based motherboards with a bootable clean install of OS X.  The patched AppleHDA.kext only works with the codec the kext was edited for and replaces the native AppleHDA.kext.

toledaALC885_patch supports
1. Mountain Lion 10.8 - 10.8.4

Supported Audio-IDs (Audio_ID = layout-id)
1. Audio_ID: 1 - 5/6 port ALC885 onboard audio and AMD/Nvidia HDMI audio*
Audio_IDs: 1 supports analog 5.1 surround sound, 3 does not
Audio_IDs: 1 requires HDMI audio dsdt edits for HDMI audio.

Requirements
1. Native S/L/E/AppleHDA.kext (use OS X combo update to restore native AppleHDA.kext)
2. Audio_ID injection enabled with one method below
2a. S/L/E/HDAEnabler1.kext or S/L/E/HDAEnabler2.kext
2b. dsdt/HDEF/layout-id=Audio_ID
2c. Clover with Config.plist/PCI/HDAInjection=Audio_ID
3. Realtek ALC885 installed on motherboard.

Download
1. https://github.com/toleda/audio_ALC885
2. Select: Download ZIP

Recommendation
1. Archive Desktop/audio_ALC885 folder as audio_ALC885_v10.x.x before patching a new AppleHDA.kext.  Patch must be run after every Software Update/Combo Update.

ALC885 Patch Procedure
Open Applications/Utilities/Terminal
One at a time, copy each command below, Paste into the Terminal window and press Enter
The ALC885 Patch makes a backup of the original AppleHDA.kext (AppleHDA-orig.kext) and a backup of the installed patched ALC885 AppleHDA.kext (AppleHDA.kext) available in Desktop/audio_ALC885. Uncommented Terminal commands below.

I.	Patch preparation/Backup installed AppleHDA.kext/Fix patch file permissions

cp -R Downloads/audio_ALC885-master Desktop/audio_ALC885
cp -R /System/Library/Extensions/AppleHDA.kext Desktop/audio_ALC885/AppleHDA-orig.kext
cd Desktop/audio_ALC885
unzip 885.zip
cd 885

II.	Install ALC885 layouts, pathmaps and pin configurations, select one layout.xml

sudo install -m 644 -o root -g wheel info.plist /System/Library/Extensions/AppleHDA.kext/Contents/Plugins/AppleHDAHardwareConfigDriver.kext/Contents/info.plist

sudo rm -R /System/Library/Extensions/AppleHDA.kext/Contents/Resources/layout*

sudo install -m 644 -o root -g wheel Platforms.xml.zlib layout1.xml.zlib  /System/Library/Extensions/AppleHDA.kext/Contents/Resources

III.	Install codec binary patch  (Native, no binary patch required)
1. For ML 10.8 - 10.8.4:

IV.	Fix permissions and backup Patched ALC885 AppleHDA.kext

sudo chown -R root:wheel /System/Library/Extensions/AppleHDA.kext
sudo touch /System/Library/Extensions

cd ~
rm -R Desktop/audio_ALC885/885
cp -R /System/Library/Extensions/AppleHDA.kext Desktop/audio_ALC885/AppleHDA.kext

exit

V. 	Restart

VI.	ALC885 audio verification

1. Open System Preferences/Sound/Output and Select Internal Speakers
2. Connect speakers or headphones to the Green motherboard audio port
3. Open iTunes, verify output device is Computer, select a song and Press Play

VII	Troubleshooting

1. Verify S/L/E/AppleHDA.kext installed and version
2. Verify IOReg/HDEDF/Layout-id is 1, 2 or 3
3. Verify permissions: run Disk Utility/Select system disk/Repair Permissions and Restart

VIII.	Problem Reporting

1. Motherboard/BIOS version/Graphics/OS X version
2. Guide used, steps completed, problem description
2. Copy of IOReg
